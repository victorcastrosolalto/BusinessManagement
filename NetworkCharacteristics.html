4. Characteristics of Computing Networks 📏💨🔒

Standards, capacity, and security are key considerations for network functionality.

4.1 Network Standards and Protocols

📜

Network Protocol

A set of agreed rules for formatting, sending, and receiving data.

🔗

Application Layer

Protocols used by applications like web browsers (e.g., HTTP).

📦

Transport Layer

Breaks data into packets and reassembles them, using protocols like TCP.

🗺️

Internet Layer

Adds IP addresses and routes packets to their destination.

🔌

Physical Network Layer

Allows different network technologies to work together (e.g., Ethernet).

🅰️➡️🔢

Domain Name Server (DNS)

Translates human-readable domain names into IP addresses for routing.

4.2 Network Capacity

🛣️

Bandwidth

The maximum rate of data transfer at any one time, measured in Hertz (Hz).

🚀

Speed

The time it takes for data to be transferred, measured in Mbps.

🗜️

Data Compression

Reduces file size for efficient storage. Lossy removes data, Lossless does not.

⚖️

Net Neutrality

The principle that all data should be treated equally by ISPs, without giving preferential treatment.

4.3 Security

🔑

Authentication

Basic security using unique usernames and passwords to verify a user's identity.

📝

Access Rights

Controls which files and resources a user can access and with what permissions.

🔒

Multi-factor Authentication

Uses multiple methods like biometrics or physical tokens to verify a user's identity for greater security.

🛡️

Firewall

Hardware or software that blocks unauthorized network access by inspecting incoming and outgoing traffic.

🕵️

Proxy Server

Acts as an intermediary to forward requests, masking the user's actual IP address and filtering content.

🔐

VPN

Secures public internet connections by creating a private network, often used for remote access.

🔏

Encryption

A process of securing data on networks by scrambling it into an unreadable format.
